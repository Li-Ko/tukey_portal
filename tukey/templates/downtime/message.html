We opened the Protected Data Cloud Pre-Pilot Project just over a year ago in February 2013 and during this time we have developed a new architecture (PDC v2.0) that we will be rolling out over approximately the next month.

 

To transition from current architecture (PDC v1.0) to PDC 2.0 will require powering down and re-imaging the hardware, which means unfortunately that all data, virtual machines, and configuration files on the current PDC v1.0 system must be saved on another resource or they will be lost in the transition to the new architecture.

 

The PDC v2.0 architecture will include OpenStack’s Swift object store (which corresponds to Amazon’s S3) and OpenStack’s Cinder block storage (which corresponds to Amazon’s EBS). Both of these systems have proved stable to the sizes we require (1-20 PB). An advantage of this approach is that pipelines developed for this architecture can more easily be ported to Amazon’s AWS.

 

The new PDC v2.0 architecture will not use the GlusterFS as the primary file system. We have had a number of tickets that relate directly or indirectly to our use of GlusterFS, and we expect the new PDC 2.0 storage system to have fewer problems.

 

The PDC 1.0 system will be down for approximately two weeks as we rebuild the system as a PDC v2.0 system. Select Beta testers will be given access to the PDC v2.0 system to help with testing.   The PDC is expected to be available again to all users on June 16, 2014.

 

What follows are next steps that require action on your part.  

 

Action Required:

On May 12, 2014, compute will be turned off on the PDC v1.0 system and it will only be used to transfer files. On May 19, 2014, PDC v1.0 users will be locked out and the rebuild will begin in full, which means prior to that date all user data must be stored off-site.   We highly recommend you back up your data, programs and configuration files off site using your own storage devices.   For users unable to provide backup themselves, we can provide space for users that require it, but ask that you specifically contact us at support@opensciencedatacloud.org by May 12, 2014. PDC users should take the following steps for each object type:

 

Files:

  • Review and identify any files you need.
  • Back up files to your own backup environment outside the PDC.

 

Instances:

  • Current instances from the PDC will not be transferrable.
  • We recommend that you create brand new instances in the new PDC environment using the base images that will be provided. 
  • We will be creating a standard Bioinformatics image using the software packages in regular use.   If you have suggestions about particular packages you'd like installed please contact us at support@opensciencedatacloud.org  

 

General Timeline Estimates:

  • PDC v1.0 System Compute off: May 12, 2014
  • PDC v1.0 System Closed: May 19, 2014
  • PDC v1.0 System Rebuild Complete:  June 2, 2014
  • BETA Users given access to PDC v2.0 System:  June 2, 2014
  • PDC v2.0 System Reopened to all users: June 16, 2014

Thank you for your continued patience during this upgrade.   If you have any questions about these changes or instructions, please contact support@opensciencedatacloud.org.
